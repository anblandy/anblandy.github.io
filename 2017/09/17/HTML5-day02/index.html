HTML5第二天·CSS3

  • CSS3兼容性差,需要添加浏览器私有前缀
  • 移动端用的多,浏览器版本高
  • 在不断改进中
  • 应用相对广泛

如何对待CSS3

  • 坚持渐进增强原则(another.优雅降级)

##选择器##

  1. 基本选择器(C2)
  2. 关系选择器(C2)
  3. 属性选择器[用在表单元素中居多],[有兼容性问题]
  4. 伪类选择器
    1. :nth-child( num ) ( 起始为 1 ,参数为”odd”时选择奇数,参数为”even”时选择偶数,也可以用”2n”,”2n+1”,”3n”这一类表达式 )
    2. :first-child() [ C2 ]
    3. :last-child() [ C3 ]
    4. :nth-of-type() [ first-of-type , last-of-type ] (参数可以是数字, odd, even, n的表达式 ) ( nth-of-type比nth-child选择条件上宽松 )
    5. 表单相关元素
      1. input:focus
      2. input:checked
      3. input:enabled
      4. input:disabled
      5. E:target [ 锚点跳转时选中 ]
  5. 伪元素选择器
    1. 渲染一个虚拟的标签
    2. DOM树上不存在,不能通过JS获取,可以利用切换类名去更改样式
    3. 单标签没有伪元素
    4. 伪元素默认是行内元素
    5. 使用伪元素清除浮动是因为伪元素不占用DOM内存
    6. ::双冒号是为了与伪类区分,但是为了兼容IE6,7,8,使用 : 单冒号, 手机端可以使用 ::
    7. 伪元素需要hover, 使用父元素:hover:after( before )格式

      icon-font

      font-class方式引用时直接到icon-font css 配置文件里提取伪类样式内的代码

Web字体

  1. 兼容性:支持所有浏览器
  2. 不失真,矢量
  3. 改颜色方便
  4. 阿里 : http://www.iconfont.com
  5. 有字库: http://www.youziku.com
  6. 插入在线的文件引用和提供的类
  7. 不适合大面积使用,涉及性能和可读性问题

边框圆角 border-radius

  1. 只有一个值的情况下,四个角的圆的水平和垂直半径都是同一个值
  2. 两个值是(左上和右下),(右上和左下)
  3. 三个值的情况下(左上),(右上和左下),(右下)
  4. 四个值的情况下,左上,右上,右下,左上
  5. 单独控制X和Y方向半径 border-radius : X方向半径 / Y方向半径
  6. 分别控制四个方向上的水平半径和垂直半径
    border-radius: 10px 20px 30px 40px /40px 30px 20px 10px
  7. 单位可以是像素,百分比

    阴影 box-shadow

  8. W3C坐标系原点在左上角,水平正方向朝右,垂直正方向朝下
  9. 前两个参数控制阴影的水平,垂直的偏移,第三个值控制阴影羽化半径(默认值为0),第四个参数控制阴影的大小(默认值为0),第五个参数为颜色,第六个参数控制内外阴影( inset / outset )
  10. 颜色单位一般使用rgba
  11. outset 是默认值,不能写,写了阴影不显示
  12. 可以写多个阴影

    文字阴影 text-shadow

    同于box-shadow

背景系列

  1. 背景是否随着页面滚动而滚动 background-attachment

    backgorund-attachment : scroll / fixed;

  2. 背景平铺的原点 background-origin

    background-origin: padding-box(默认值) / content-box / border-box
    1. padding-box : padding的0,0点开始平铺
    2. content-box : content的0,0点开始平铺
    3. border-box : border的0,0点开始平铺

  1. 多重背景
    在background后写多次”url() x y no-repeat / repeat”即可引入多张背景图片

  2. 控制背景图片大小 background-size

    1. 一个参数时,会设置给图片宽度,高度会自动缩放
    2. 两个参数时,会分别设置给宽度,高度,图片可能会产生变形
    3. 参数为百分比时,参照的是盒子的宽高,不是图片自身大小
    4. 参数为contain时将图片拉伸到有一条边触碰到盒子边缘时就停止放大
    5. 参数为cover时,contain的基础上,将短的那一边也拉伸到盒子边缘,并自动裁切超出的部分,不留白
